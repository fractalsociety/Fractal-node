M4 Update

What I just implemented

  • Receipt logs: eth_getTransactionReceipt now returns a logs array of full log objects (same shape as eth_getLogs), including blockHash. logIndex is block-scoped (sum of prior logs in the block before this tx).
  • Real EVM CALL execution (devnet): fractal-evm now runs stored bytecode via revm and commits storage writes.
  • Persistent EVM storage: added State.evm_storage (slot map) and wired it into the EVM database bridge.
  • RPC storage reads: eth_getStorageAt now returns real values from State.evm_storage.
  • Contract bytecode eth_call path verified: added an automated test proving a contract CALL can SSTORE and RETURN 32 bytes deterministically.
  • eth_getLogs filters: topics (null wildcards, exact 32-byte hashes, OR-arrays per position), blockHash single-block mode (exclusive with fromBlock/toBlock), address as one address or an array (OR), block tags earliest / pending / latest / hex.
  • CREATE (revm): TxBody::EvmCreate uses top-level TxKind::Create; deployed code is init RETURN output; gas/logs/receipt bloom follow the same path as EvmCall; deployer nonce comes from revm (no double bump).
  • logsBloom: standard 2048-bit Ethereum bloom on receipts and merged block bloom.
